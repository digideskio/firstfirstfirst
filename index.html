First first first

The web/app universe has a lot to say about what to do first when re-building/re-designing or embarking on a new technical adventure. Turns out almost everybody is right. There are a number of things that need to be done first. You work in software development so you understand this double think norm. Yet when everything is urgent, nothing is (insert meme here). This means we can then relax and make a list.

First Why
Readme First Fail first's little brother. Let your own documentation guide you.
Fail First If you write them (tests), the code will come... out of sheer annoyance.
Cloud First Nobody wants to be in the hosting business.
Mobile First The classic that started it all. Phones are gonna be huge!
Data First Think analysis early. Eventually you'll need what's going on.
German First Say it with me: "internationalization." It's a long word without a line break.
Ugly First Your users don't necessary create the best things. Like German text.
Personalization First People want different things. You can't afford to be boring.
Offline First Mobile first's cousin. Stuff breaks.
